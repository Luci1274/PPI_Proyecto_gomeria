# Caso de uso: Registrar Compra
# Actores: Administrador
# Precondiciones: El proveedor y los productos deben estar registrados en el sistema.

# Caminos Basico:
1. El administrador ingresa proveedor.
2. El administrador agrega productos, cantidad, precio unitario y proveedor.
3. El sistema valida los datos.
4. El sistema visualiza la informacion ingresada.
5. El administrador guarda los datos.
6. El sistema actualiza stock y guarda registro.
7. El sistema visualiza listado de compras.
8. El administrador modifica los datos ingresados.
9. El sistema valida los datos.
10. El sistema valida correctamente y visualiza los cambios en el listado. 

# Caminos Alternativos:
2.A Datos del registro no valido.
2.A.1 Vuelve al paso 1.
7.A Datos modificados no valido.
7.A.1 Vuelve al paso 6.

# Postcondiciones: Compra registrada en el sistema.
